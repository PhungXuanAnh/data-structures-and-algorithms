The Boundary Concept

[...elements < pivot...] [...elements >= pivot...] [unprocessed] [pivot] left smaller_upper +1 index-1 index right boundary
The invariant (rule) we maintain:
‚Ä¢ From left to smaller_upper_boundary: all elements < pivot
‚Ä¢ From smaller_upper_boundary + 1 to index - 1: all elements >= pivot
‚Ä¢ From index to right - 1: not yet examined
‚Ä¢ At right: the pivot itself

Why Start at -1?

When smaller_upper_boundary = -1:
‚Ä¢ The "smaller than pivot" section spans from index 0 to -1
‚Ä¢ That's an EMPTY range! üéØ
‚Ä¢ Perfect! We haven't found any smaller elements yet!

When we find the first smaller element:
‚Ä¢ We increment: smaller_upper_boundary++, so it becomes 0
‚Ä¢ We swap it to position 0
‚Ä¢ Now the "smaller" section contains indices 0 to 0 (one element) ‚úì
